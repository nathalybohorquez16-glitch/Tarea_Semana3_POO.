## Descripción del proyecto

Este proyecto desarrolla un sistema básico de gestión de mascotas utilizando dos enfoques diferentes de programación en Python:

### 1. Programación Tradicional

Se utilizan funciones, variables y entrada de datos por teclado para registrar y mostrar la información de una mascota.

Datos registrados:
- Nombre
- Especie
- Edad

Archivo:
- programacion_tradicional/tradicional.py

---

### 2. Programación Orientada a Objetos

Se implementa una clase llamada Mascota para representar una mascota mediante atributos y métodos.

Atributos:
- nombre
- especie
- edad

Métodos:
- mostrar_informacion()
- hacer_sonido()

Archivos:
- programacion_poo/mascota.py
- programacion_poo/main.py

---

## Reflexión

La Programación Tradicional permite resolver problemas mediante funciones y variables, siendo adecuada para programas sencillos. Sin embargo, cuando el proyecto aumenta de tamaño, la organización y reutilización del código puede resultar más compleja.

La Programación Orientada a Objetos permite modelar entidades reales mediante clases y objetos. Este enfoque facilita la organización, reutilización y mantenimiento del código, haciendo que los programas sean más escalables y fáciles de comprender.

Gracias a esta actividad se pudo comparar ambos enfoques y comprender las ventajas que ofrece la Programación Orientada a Objetos para el desarrollo de software.
